# NetworkAnalytics

## Обзор

Это приложение использует несколько вспомогательных классов и служб для управления журналированием, запросами сервера и взаимодействиями с пользовательским интерфейсом. Ниже приведены описания ключевых компонентов:

### LoggingHelper

Класс `LoggingHelper` определяет структурированный формат для регистрации информации в приложении. Он содержит свойства для типа сообщения, уровня опасности, содержимого сообщения, описания и необязательного кода состояния HTTP.

### ConfigurationHelper

Класс ConfigurationHelper управляет параметрами конфигурации приложения. Он содержит методы для сохранения и загрузки данных конфигурации, в частности, связанных с IP, портом и токенами аутентификации.

### MainHttp

Класс `MainHttp` обрабатывает HTTP-запросы к серверу. Он включает методы для различных типов HTTP-запросов, таких как GET, POST, PUT и DELETE, а также обработку ошибок и управление токенами.

### CaptchaSolverWindow

`CaptchaSolverWindow` представляет собой окно, используемое для решения задач с использованием CAPTCHA. Оно интегрируется с классом `MainHttp` для отправки ответов и управления процессом решения CAPTCHA.

## Функции

- Структурированное ведение журнала с помощью класса `LoggingHelper`
- Управление конфигурацией с помощью `ConfigurationHelper`
- Обработка HTTP-запросов с помощью класса `MainHttp`
- Функция распознавания КАПЧИ с помощью "CaptchaSolverWindow"
